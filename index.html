08今すぐやるべき10のこと

Claude Code + 開発体制の最適化提案

1. Mac mini を「夜間ワーカー」にする

Mac miniにClaude Codeをインストールし、claude -p "タスク" --allowedTools Edit,Write,Bash でヘッドレス実行。GitHub Issueに「寝る前タスク」を書いておき、cronで起動 → 朝にPRが立っている状態を作る。SSH経由で外出先からも指示可能。

2. スマホ連携: GitHub Mobile + Termius

GitHub Mobileアプリで外出先からPRレビュー・Issue作成。Termius（iOSアプリ）でSSH接続し、Mac miniのClaude Codeにスマホから指示を出す。Slack Botにコマンドを送って開発タスクをキックする仕組みも作れる。

3. Claude Code の Agent Teams 機能を活用

claude agents でリサーチ・コーディング・テストを並列実行。例: 「CRMの全APIエンドポイントにテストを書いて」と指示すれば、Agent Teamsが並列にテスト生成。1つの指示で複数ファイルを同時に修正できる。

4. CLAUDE.md をプロジェクトルートに設置

各プロジェクトのルートに CLAUDE.md を置くことで、そのプロジェクト固有のルール・構造をClaude Codeに毎回教えなくて済む。現在のホームディレクトリのCLAUDE.mdは素晴らしいが、プロジェクト別にも作ると精度が上がる。

5. テストを書く習慣をつける (テスト0件 → まず10件)

"このAPIエンドポイントのテストを書いて" とClaude Codeに指示するだけでOK。最初は主要なCron/Webhook APIの統合テストから。壊れやすい箇所（ステージ自動遷移、売上計算）を優先。

6. /review コマンドで自己レビュー

コミット前に claude /review を実行し、セキュリティ問題・パフォーマンス問題・バグを自動検出。19件のHIGHバグは事前に防げた可能性がある。Git hooksに組み込むことも可能。

7. 「作業のムラ」を解消する構造

GitHub Issuesにタスクを全て書き出し → 優先度ラベル (P0/P1/P2) を付ける → 1日3タスクを目標。PCの前にいない時間も、スマホからIssueのトリアージやPRレビューができる。作業は「やる/やらない」ではなく「何をやるか」の選択に。

8. OpenCrawler (Mac mini) の活用計画

Mac miniでOpenCrawlerを常時稼働させ、競合サイト・求人データ・業界ニュースを自動収集 → CRMのマーケ分析ダッシュボードに統合。AIbouのBrainMatrixに外部データを流し込み、CEOの意思決定を支援。

9. Sentry + 構造化ログの導入

console.log 121箇所 → pino (構造化ログ) + Sentry (エラー監視) に置き換え。本番環境のエラーがSlackに即通知される体制を構築。これまでの「バグ発見→修正」サイクルを「バグ検知→自動通知→修正」に。

10. Claude Code Hooks で開発フロー自動化

Claude Code の Hooks 機能で、ファイル保存時に自動lint、コミット前に自動テスト、デプロイ前にセキュリティチェックを設定。.claude/hooks.json に定義するだけで、開発品質が自動的に担保される。
