1965年創業、ランドセル製造から始まり、日本を代表するD2Cレザーブランドへ成長。「品質と物語」で勝負するブランド経営の先駆者。

企業プロフィール

  • 正式名称: 株式会社 土屋鞄製造所
  • 創業: 1965年（昭和40年）
  • 代表: 土屋成範（3代目）
  • 本社: 東京都足立区（西新井工房）
  • 事業: 鞄・革小物の企画・製造・販売
  • 販路: 直営店31店舗 + 自社EC（Shopify）
  • 特徴: 中間マージンゼロの完全D2Cモデル
  • ブランド: 土屋鞄、grirose、depsoa、objcts.io

商品ラインナップ

  • バッグ: ビジネス、トート、ショルダー、バックパック（324 SKU）
  • 財布: 長財布、二つ折り、コンパクト（160 SKU）
  • 革小物: 名刺入れ、キーケース、ペンケース等（237 SKU）
  • ランドセル: 3ブランド展開、約110モデル
  • objcts.io: テック系レザーブランド（PC/タブレットケース等）
  • リユース事業: 自社製品の買取・再販
  • オトナランドセル: 大人向けランドセル型バッグ
  • ギフト: 名入れ・刻印サービス対応

なぜ土屋鞄がマーケ分析SaaSの理想的なパートナーなのか

D2C完全直販モデルにより、顧客接点からCVまでの全データを自社で保有。Shopify+Klaviyoの標準的なスタックを使用しており、 同業他社への水平展開が容易。さらに「ランドセル」と「大人向け革製品」という全く異なる購買行動を持つ2事業を運営しており、 分析ツールの汎用性を証明する最良のテストケースとなる。

Section 03

技術スタック

Shopifyを中心に、分析・CRM・UXツールを統合した堅実なスタック構成。SaaS連携の土台として極めて標準的。

S

Shopify (Prestige v10.0.2)

EC基盤。Prestige テーマはハイエンドD2C向け。7,672 SKU を運用。Shopify Plus の可能性あり。

G

Google Tag Manager (GTM-9MQ6)

タグ管理の中核。GA4、Google Ads、Meta Pixel 等の全トラッキングタグを統合管理。

K

Klaviyo

メールマーケティング / CRM。セグメント配信、フロー自動化、購買データ連携。開封率30-40%。

C

Microsoft Clarity (dxmcutx5p9)

ヒートマップ・セッション録画。無料でUX分析。スクロール深度、クリック分布、レイジクリック検出。

J

Judge.me

商品レビュープラットフォーム。写真レビュー、Q&A機能。SEO向けリッチスニペット出力。

P

Apple Pay / Google Pay / Shopify Payments

1クリック決済で購入CVR向上。カート離脱率の削減に直結する決済インフラ。

W

hulkapps Wishlist

ウィッシュリスト機能。非購入時のマイクロCV。リマインドメールとの連動でCV率向上。

I

Inventory Info App

在庫情報管理。在庫切れ通知、入荷予定表示。需要予測データの基盤。

SaaS連携観点での評価

Shopify + Klaviyo + GTM は日本のD2C/ECブランドの「標準スタック」であり、 このスタックに対応するダッシュボードSaaSは極めて水平展開しやすい。 Shopify Admin API でほぼ全データを取得可能で、Klaviyo API も充実。 Microsoft Clarity はデータエクスポートが限定的だが、Clarity API（プレビュー）を活用する余地がある。

「売らない投稿」戦略の本質

土屋鞄のInstagram戦略の核心は、直接的な販促投稿を極力避け、 革素材の美しさ、職人の手仕事、製品が使われるライフシーンにフォーカスしたコンテンツを発信し続けること。 これにより「憧れ」と「信頼」のブランドエクイティを構築し、購入検討時の第一想起を獲得する。 広告ROASでは測れない、ブランド資産の積み上げ。

SaaS としてこの「ブランド指標」をどう定量化するかが差別化ポイント。 指名検索ボリューム、UGC数、NPS推移をダッシュボード化する価値がここにある。

Section 07

マーケ分析ダッシュボードの提案

3段階のTier構成で、導入ハードルを下げつつ、段階的に分析深度を上げていく設計。

Tier 1 / 必須

Daily Dashboard

毎朝チェック / リアルタイム

  • 売上ダッシュボード — 日別/週別/月別売上、前年比、目標進捗
  • チャネル別ROAS — Google/Meta/Organic の費用対効果リアルタイム
  • 商品別パフォーマンス — 売れ筋TOP20、在庫アラート、カテゴリ別推移
  • CVRファネル — 訪問→PDP→カート→購入の各ステップ転換率
  • カート離脱分析 — 離脱ポイント特定、リカバリーメール効果
Tier 2 / 週次

Growth Analytics

週次レビュー / 施策効果測定

  • コホート分析 — 月別獲得コホートの継続率・LTV推移
  • LTV予測 — 購入回数ベースの将来LTV予測モデル
  • RFM分析 — Recency/Frequency/Monetary セグメント
  • クリエイティブ分析 — 広告クリエイティブ別のCTR/CVR/ROAS
  • Klaviyoフロー効果 — 各自動メールフローのCV貢献
Tier 3 / 月次

Strategic Insights

月次経営会議 / 中長期戦略

  • ブランド指標 — NPS、指名検索推移、UGC数、メディア露出
  • 競合ベンチマーク — SimilarWeb/SEMrush データ横断比較
  • 季節性予測 — 過去データベースの需要予測（特にランドセル）
  • カテゴリ別マージン — 商品カテゴリごとの粗利率分析
  • 顧客クラスター分析 — AIベースのペルソナ自動分類

ダッシュボード画面構成（ワイヤーフレーム概要）

トップページ

  • 今日の売上サマリー（前日比、前年同日比）
  • 主要KPIカード（CVR、AOV、セッション数）
  • チャネル別売上パイチャート
  • 直近7日の売上折れ線グラフ
  • アラート（異常値検知、在庫切れ等）

深掘りページ

  • 商品詳細: SKU別売上、在庫、レビュー
  • 顧客詳細: RFMマトリクス、コホート
  • 広告詳細: キャンペーン別/クリエイティブ別
  • CRM詳細: Klaviyoフロー効果、セグメント
  • レポート: 自動生成PDF、Slack通知

差別化ポイント: 「朝のSlack通知」機能

毎朝8時にSlack/LINEで昨日のサマリーを自動通知。 「昨日の売上: 320万円（前日比+12%）、CVR: 2.1%、在庫アラート: 3件」のような 1行サマリー。ダッシュボードを開かなくても状況を把握できる。 これだけでも月額5万円の価値がある機能。

技術実装の方針

Shopify Webhooks によるリアルタイムデータ同期 + 日次バッチでの集計処理を併用。 Webhooks で orders/create, orders/updated, products/update 等のイベントをキャッチし、 分析用DBに蓄積。Klaviyo は API ポーリング（6時間間隔）でメトリクスを同期。 フロントエンドは Next.js + Recharts/Nivo でダッシュボードを構築。

想定Q&A（緑川さんから聞かれそうな質問）

Q: 「Shopify Analyticsと何が違うの？」

A: Shopify Analyticsは単一チャネルのデータのみ。我々は Google Ads、Meta、Klaviyo、Judge.me を統合し、 「どのチャネルで獲得した顧客が最もLTVが高いか」「どのクリエイティブが最もROASが高いか」を一画面で可視化します。

Q: 「データのセキュリティは？」

A: Shopify APIの read-only 権限のみ使用。顧客の個人情報はハッシュ化して保存。 SOC2 Type II 取得に向けて準備中。データは日本リージョン（AWS ap-northeast-1）に保存。

Q: 「Looker Studioで十分では？」

A: Looker Studioは汎用BIなので、Shopifyデータの取り込みにコネクタ設定が必要で、 ECに特化したKPI（CVRファネル、RFM、コホートLTV等）を出すには相当なカスタマイズが必要。 我々は Shopify接続の瞬間からこれらが全て表示されます。セットアップ時間は5分です。

Q: 「月15万円は高くない？」

A: 現在、同等のダッシュボードをLooker Studioで構築・維持するための人件費（分析担当者の工数）を 月額換算すると、おそらく30-50万円相当。さらにTriple Whale（海外同等サービス）は月$300-$500。 日本語対応・日本EC商習慣最適化込みで月15万円は、むしろ投資対効果が高い選択です。
